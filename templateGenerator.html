Paramètres

Détermine la logique générale du parcours. Ce choix influence la structure des scènes, les types de décisions proposées et les variables utilisées. Par exemple, un parcours de gestion de crise génère plus de tensions et de contraintes qu’un parcours d’apprentissage progressif.

Contexte narratif

Indiquez le domaine dans lequel se déroule le parcours. Cela permet de générer des textes génériques cohérents. Exemples : gestion de projet, management, qualité, cybersécurité, relation client, production, formation, etc.
Indiquez le rôle incarné par le joueur dans le scénario. Cela permet de contextualiser les décisions proposées. Exemples : responsable, chef de projet, analyste, coordinateur, technicien, consultant, etc.
Décrivez ce que le joueur doit comprendre à travers le parcours. Cela servira à générer le diagnostic final. Exemple : comprendre les effets cumulés des décisions, apprendre à gérer l’incertitude, structurer une analyse avant d’agir, etc.

Structure

Détermine le nombre de scènes et la complexité globale. Un parcours court permet un test rapide. Un parcours standard offre un bon équilibre. Un parcours avancé permet d’introduire plus de tensions, de décisions intermédiaires et de corrections.
Les variables représentent l’état interne du système. Elles évoluent selon les décisions prises. Plus il y a de variables, plus le parcours devient riche mais aussi plus complexe à équilibrer.
Détermine la forme de conclusion du parcours. Une issue simple donne une seule fin. Plusieurs issues permettent différents résultats. L’option diagnostic met l’accent sur l’analyse des décisions prises.

Tensions à intégrer

Les tensions sont des difficultés qui compliquent la prise de décision. Elles permettent de rendre le parcours plus réaliste. Par exemple : pression du temps, manque d’informations, désaccords, fatigue, signaux faibles, etc. Il est conseillé de ne pas en sélectionner trop pour garder un parcours lisible.
Sélection recommandée : 2 à 4 tensions.

Types de décisions

Définit les dilemmes proposés au joueur. Chaque type produit des arbitrages réalistes entre plusieurs options ayant chacune des avantages et des inconvénients. Il est recommandé d’en sélectionner entre deux et quatre.
Sélection recommandée : 2 à 4 types.

Comment fonctionne le générateur

          

            Ce générateur crée automatiquement une structure de parcours narratif à partir d’un brief minimal. Il ne produit pas un jeu finalisé, mais
            un template JSON cohérent, directement importable dans l’outil auteur.
          

          
Étapes recommandées

          

            Choisissez d’abord le type de parcours. Renseignez ensuite le contexte, le rôle du joueur et l’objectif du parcours. Sélectionnez la
            longueur, le nombre de variables, les tensions et les types de décisions. Enfin, générez la structure puis vérifiez l’aperçu avant
            d’exporter le JSON.
          

          
Ce que produit l’outil

          

            Le générateur crée les variables, les scènes principales, les choix génériques, les règles d’effondrement et les diagnostics. Les textes
            restent génériques pour pouvoir être adaptés ensuite dans l’outil auteur.
          

          
Bon usage

          

            Pour une V1, il est conseillé de limiter le nombre de tensions et de types de décisions. Une structure trop chargée devient vite difficile
            à retravailler. Commencez simple, puis enrichissez dans l’outil auteur si nécessaire.
